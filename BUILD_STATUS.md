### Runs on any GPU (auto-configured)
Every entry point (`00_MAIN.ipynb`, `run_full.py`, `serve_app.py`) calls
`ragguard.autotune.apply()`, which **detects GPU VRAM and picks the path**, installing
`bitsandbytes` on demand:

| VRAM | Path | Peak (est.) |
|---|---|---|
| ‚â•30 GB (high-VRAM GPU) | Qwen3-8B bf16, batch 4 | ~28‚Äì32 GB |
| 20‚Äì30 GB (24 GB GPU, e.g. Colab L4) | Qwen3-8B bf16, sequential | ~18‚Äì21 GB |
| 10‚Äì20 GB (12‚Äì16 GB GPU) | Qwen3-8B **4-bit** | ~8‚Äì11 GB |
| 6‚Äì10 GB | Qwen2.5-3B bf16 | ~7‚Äì9 GB |
| <6 GB / no GPU | Qwen2.5-3B 4-bit / 0.5B CPU | ‚Äî |

Batched generation also **backs off on CUDA OOM** (‚Üí sequential). The canonical report
numbers are the **bf16 8B** run (‚â•30 GB GPU); a teammate on 4-bit/smaller model will see slightly
different ASR/utility (note it in the report if used).
